# Codex 작업 지시문

아래 규칙은 Holysong 개발 전체에서 일관적으로 적용되는 Codex 운영 규칙이다.

### [운영 규칙 요약]
- 설계/TDD 시나리오는 일반모드에서 이미 확정되었다.
- 당신은 test body 및 함수 body만 구현한다.
- 파일명/클래스명/함수명은 절대 변경하지 말 것.
- 기존 파일을 재생성하지 말 것.
- 아키텍처(Service → Repository → Model) 흐름을 변경하지 말 것.
- 새로운 규칙/기능을 만들어내지 말 것.
- 주어진 TDD 시나리오에 **직접적으로 대응하는 코드만** 작성할 것.
